Lab goal

This optional lab is mainly for CSE5006 students. You will install Apache TinkerPop Gremlin Server, start a local graph database, and run a Node.js program that creates vertices and edges. This is not required for every student, but it is useful if you want to understand why graph databases are powerful for relationships and paths.

 

Big idea

SQL focuses on tables. MongoDB focuses on documents. Graph databases focus on relationships. A graph query often asks: who is connected to whom, through what path?

Step 1: Start from the lab folder

Begin from the unzipped lab root. We store the root path in LAB_ROOT because this lab later moves to your home directory to install Gremlin Server.

cd ~/Downloads/lab-v2
LAB_ROOT="$PWD"
pwd
ls

Step 2: Install Java and unzip

Gremlin Server runs on Java. Install Java and the small tools needed to download and extract the server package.

sudo apt-get update
sudo apt-get install -y openjdk-17-jre-headless unzip curl
java -version

Step 3: Download Gremlin Server

Use Gremlin Server 3.7.6 and the matching Node.js client version. This avoids known WebSocket compatibility problems that can appear when older Gremlin clients are used with modern Node.js 22 environments. The command uses -fL so download errors fail clearly, and --retry 3 to reduce temporary network failure problems. If the folder already exists, it will not unzip again.

cd ~
if [ ! -d "$HOME/apache-tinkerpop-gremlin-server-3.7.6" ]; then
  curl -fL --retry 3 https://dlcdn.apache.org/tinkerpop/3.7.6/apache-tinkerpop-gremlin-server-3.7.6-bin.zip -o apache-tinkerpop-gremlin-server-3.7.6-bin.zip
  unzip -q apache-tinkerpop-gremlin-server-3.7.6-bin.zip
fi
ls "$HOME/apache-tinkerpop-gremlin-server-3.7.6"

Step 4: Install the Node.js dependencies

Return to the lab code folder before installing npm dependencies. The package uses gremlin 3.7.6 and a supplied package-lock.json, so npm ci installs exactly the dependency tree tested for this lab.

cd "$LAB_ROOT/lab3-optional-gremlin"
npm ci

Step 5: Start Gremlin Server

Use two terminals. Terminal 1 keeps the graph server running.

cd ~/apache-tinkerpop-gremlin-server-3.7.6
./bin/gremlin-server.sh conf/gremlin-server-modern.yaml

Leave this terminal open. If you close it, the Node.js client cannot connect to ws://localhost:8182/gremlin.

Step 6: Run the Node.js graph CRUD demo

Open Terminal 2 and run the Node.js code from the lab folder.

cd ~/Downloads/lab-v2/lab3-optional-gremlin
npm start

The connected-student query now excludes Alice herself. It asks: which other student shares an enrolled course with Alice?

Step 7: Modify the graph code yourself

Open gremlin_crud.js.

code gremlin_crud.js
  • Change Alice Chen or Emma Li to another fake name.
  • Add a new student vertex by copying one of the addV('student') lines.
  • Add a new edge from that student to the Python course.
  • Run npm start again and inspect how the connected-student query changes.

Step 8: Stop Gremlin Server

Return to Terminal 1 and press Ctrl + C. This stops Gremlin Server.

Expected outcome

By the end of this optional lab, you should be able to explain the difference between a row, a document, and a graph relationship. You should also understand why graph databases are useful when the question is about paths, connections and indirect relationships.
