Lab goal

In this lab you will install PostgreSQL and MongoDB on Ubuntu, create a small teaching dataset by hand, test CRUD operations directly in the database console, and then run Node.js code that performs the same kind of CRUD operations from an application.

This lab is deliberately step by step. Do not rush to npm start. A database-backed application only makes sense when you understand the database service, the database user, the connection string, the table or collection, and the code that sends operations to the database.

 

Important safety note

Run these commands only inside your dedicated course Ubuntu VM. This lab installs database services, changes local configuration files, and resets teaching data. Do not run it on a production server or on a machine that already has important PostgreSQL or MongoDB data.

Step 4: Create the PostgreSQL user and database

We create one local classroom user and one local classroom database. The password 123456 is only for this local lab VM. The following command is repeatable: if the role already exists, it refreshes the password; if the database already exists, it keeps it and fixes the owner.

sudo -u postgres psql -v ON_ERROR_STOP=1 <<'SQL'
DO $$
BEGIN
  IF NOT EXISTS (SELECT FROM pg_roles WHERE rolname = 'university_user') THEN
    CREATE ROLE university_user LOGIN PASSWORD '123456';
  ELSE
    ALTER ROLE university_user WITH LOGIN PASSWORD '123456';
  END IF;
END
$$;

SELECT 'CREATE DATABASE university_db OWNER university_user'
WHERE NOT EXISTS (SELECT FROM pg_database WHERE datname = 'university_db')\gexec

ALTER DATABASE university_db OWNER TO university_user;
SQL

Check that the classroom account can connect.

PGPASSWORD=123456 psql -v ON_ERROR_STOP=1 -h localhost -U university_user -d university_db -c "SELECT current_database(), current_user;"

Step 6: Manually practise PostgreSQL CRUD

Run these commands as one block. They show Create, Read, Update and Delete without Node.js. This is important because you should confirm the database works before blaming application code.

PGPASSWORD=123456 psql -v ON_ERROR_STOP=1 -h localhost -U university_user -d university_db <<'SQL'
\dt
SELECT id, name, major FROM students ORDER BY id;

INSERT INTO students (id, name, email, major)
VALUES ('S9', 'Ivy Zhang', 'ivy.zhang@example.edu', 'Testing');

SELECT id, name, major FROM students WHERE id = 'S9';

UPDATE students
SET major = 'Updated Testing Major'
WHERE id = 'S9';

SELECT id, name, major FROM students WHERE id = 'S9';

DELETE FROM students WHERE id = 'S9';

SELECT id, name, major FROM students ORDER BY id;
SQL

Step 7: Install MongoDB

MongoDB is a document database. Use the official MongoDB 8.0 Ubuntu repository. This lab assumes a fresh course VM. If you previously enabled MongoDB authentication with an unknown password, reset the VM or restore your MongoDB configuration before continuing.

sudo apt-get install -y ca-certificates curl gnupg
. /etc/os-release
curl -fsSL https://pgp.mongodb.com/server-8.0.asc -o /tmp/mongodb-server-8.0.asc
gpg --batch --yes --dearmor -o /tmp/mongodb-server-8.0.gpg /tmp/mongodb-server-8.0.asc
sudo mv /tmp/mongodb-server-8.0.gpg /usr/share/keyrings/mongodb-server-8.0.gpg
echo "deb [ arch=amd64,arm64 signed-by=/usr/share/keyrings/mongodb-server-8.0.gpg ] https://repo.mongodb.org/apt/ubuntu ${UBUNTU_CODENAME}/mongodb-org/8.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-8.0.list
sudo apt-get update
sudo apt-get install -y mongodb-org
sudo systemctl enable mongod
sudo systemctl start mongod
sudo systemctl is-active --quiet mongod && echo "MongoDB is running"

If MongoDB does not start, inspect its service log before continuing.

sudo systemctl status mongod --no-pager
sudo journalctl -u mongod -n 50 --no-pager

Step 8: Create or verify the MongoDB user

This block works on both a fresh course VM and a VM on which you have already completed this lab. It first tests the authenticated classroom account. On a fresh VM, that test is expected to fail, after which the account is created before authorisation is enabled.

Run the whole block as one command. Do not run only the middle lines separately.

if mongosh "mongodb://university_user:123456@localhost:27017/admin?authSource=admin" \
  --quiet --eval 'db.runCommand({ ping: 1 })' >/dev/null 2>&1; then
  echo "Existing authenticated classroom account is working."
else
  echo "No working classroom account was found. Creating it now."
  mongosh --quiet <<'MONGO'
use admin
db.createUser({
  user: "university_user",
  pwd: "123456",
  roles: [{ role: "readWrite", db: "university_db" }]
})
MONGO
fi

If the account test succeeds, the creation section is skipped. If this is a fresh VM, the first authentication error is expected and the block continues automatically.

Step 9: Enable MongoDB authorisation safely

This block makes a timestamped backup and avoids adding duplicate authorization lines.

sudo cp /etc/mongod.conf "/etc/mongod.conf.backup.$(date +%Y%m%d-%H%M%S)"
if ! grep -q '^security:' /etc/mongod.conf; then
  printf '\nsecurity:\n  authorization: enabled\n' | sudo tee -a /etc/mongod.conf
elif grep -q '^[[:space:]]*authorization:' /etc/mongod.conf; then
  sudo sed -i 's/^[[:space:]]*authorization:.*/  authorization: enabled/' /etc/mongod.conf
else
  sudo sed -i '/^security:/a\  authorization: enabled' /etc/mongod.conf
fi
sudo systemctl restart mongod
sleep 3
sudo systemctl is-active --quiet mongod && echo "MongoDB restarted with authorisation enabled"

db.studentProfiles.updateOne(

Step 12: Prepare and run the Node.js code

Now move into the lab folder, install the fixed npm dependencies, and run each direct-driver demo. The dependencies are pinned in package.json and locked in package-lock.json, so npm ci installs the tested dependency tree.

cd ~/Downloads/lab-v2/lab1-direct-postgresql-mongodb
cp .env.example .env
npm ci
npm run start:postgres
npm run start:mongo

The PostgreSQL program uses INSERT ... ON CONFLICT. The MongoDB program uses updateOne(..., { upsert: true }). Both are intentional: they make the CREATE step repeatable by refreshing the demo record if it already exists.

Step 14: Modify the code yourself

Use Ubuntu's built-in nano editor. Open the PostgreSQL program first:

nano postgres_direct_crud.js

In nano, use the arrow keys to move. Press Ctrl + O, then Enter to save. Press Ctrl + X to exit. To edit the MongoDB program, run:

nano mongodb_direct_crud.js
  • Open postgres_direct_crud.js. Change Grace Huang to your own fake student name, then run npm run start:postgres again.
  • Comment out the DELETE block in postgres_direct_crud.js, run the file, then check PostgreSQL manually with SELECT id, name, major FROM students ORDER BY id;.
  • Open mongodb_direct_crud.js. Change the demo course added with $addToSet, then run npm run start:mongo again.
  • After testing, restore the DELETE blocks so the demo remains repeatable.
  • Run npm run start:postgres and npm run start:mongo once more after restoring the code. Then repeat the Step 13 verification and confirm that all six results are PASS.

Expected outcome

By the end of Lab 1, you should be able to explain how PostgreSQL tables differ from MongoDB documents, how to manually test a database before using Node.js, and how direct database drivers send CRUD operations from application code.
